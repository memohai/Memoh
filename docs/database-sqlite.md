# SQLite Database Track

This document records the implementation track for adding SQLite beside the
existing PostgreSQL backend.

## Layout

- PostgreSQL migrations: `db/postgres/migrations`
- PostgreSQL queries: `db/postgres/queries`
- PostgreSQL sqlc output: `internal/db/sqlc`
- SQLite migrations: `db/sqlite/migrations`
- SQLite queries: `db/sqlite/queries`
- SQLite sqlc output: `internal/db/sqlitesqlc`

## Type Mapping

| PostgreSQL | SQLite target |
| --- | --- |
| `UUID` | `TEXT`, generated by Go |
| `JSONB` | `TEXT` JSON, optionally guarded by `CHECK(json_valid(...))` |
| `TIMESTAMPTZ` | One project-wide representation: ISO-8601 `TEXT` or Unix milliseconds `INTEGER` |
| `TEXT[]` | JSON `TEXT` array |
| `user_role` enum | `TEXT CHECK (...)` |
| `gen_random_uuid()` | Go-side UUID generation |

## Query Rewrite Hotspots

- `messages.sql`: JSONB functions, `ILIKE`, `ANY(uuid[])`, UUID/time casts.
- `token_usage.sql`: `date_trunc`, JSONB extraction, aggregate casts.
- `session_info.sql`: `jsonb_array_elements`, `jsonb_typeof`.
- `channels.sql`: `jsonb_build_object`, JSONB merge operators.
- `settings.sql`: typed nullable args such as `sqlc.narg(...)::uuid`.
- `conversations.sql`: `::jsonb`, `::uuid`, typed nulls.
- `users.sql`: enum casts, `ILIKE`, JSONB defaults.
- `channel_identities.sql`: `FOR UPDATE`, `ILIKE`, upsert differences.
- `bind.sql`: `FOR UPDATE` and transactional consume semantics.
- `media.sql`: `ANY(uuid[])`.
- `mcp_oauth.sql`: array columns and upsert behavior.
- `tool_approval.sql`: JSON, UUID, and upsert behavior.

## Verification Matrix

Run the same store contract tests against both implementations once SQLite
stores exist.

| Area | PostgreSQL | SQLite |
| --- | --- | --- |
| Store contracts | Existing sqlc-backed stores | SQLite sqlc-backed stores |
| Migrations | `db/postgres/migrations` through postgres migrate driver | `db/sqlite/migrations` through sqlite migrate driver |
| Startup | `driver = "postgres"` creates admin and starts server | `driver = "sqlite"` creates admin and starts server |
| Transactions | `BindCodeStore.Consume`, `WorkspaceStore.RecordSnapshotVersion`, route create race | Same behavior via `BEGIN IMMEDIATE` or store-local locking |
| Query semantics | message search, token usage, settings upsert, identity search | Same expected rows and pagination |

## Current Status

The repository has the directory layout and Go-facing database/store contracts.
The SQLite schema, SQLite queries, and concrete SQLite stores are intentionally
not wired into `sqlc.yaml` yet, so regular PostgreSQL generation remains stable.
