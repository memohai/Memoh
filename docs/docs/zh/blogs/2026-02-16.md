# Memoh 介绍 —— 为何需要一个常驻、容器化的家庭 Agent

## 概述

进入 2026 年，我们依然面对同一个矛盾：模型每个季度都在变强，但"Agent 体验"仍然在上下文、延迟、隐私和真实工作流上反复折戟。过去一年里，我们一直在围绕三个问题打转：

- Agent 的能力边界到底在哪里？
- 长上下文的真实价值是什么？
- 什么样的硬件形态能让"常驻个人 AI"变得自然？

Memoh 是我们把这些问题变成可构建之物的一次尝试——不是宣言，而是一个能在现实中活下去的系统。

## 故事

时间跑得很快。在"我会记住这个"和"等等，当初我们为什么这么决定的？"之间，一年就消失了。

这是构建过程中最烦人的地方：大多数进展在发生的时候根本感觉不像进展。它只是一串小决定、做了一半的线头、深夜的修复，以及偶尔真正咬合的瞬间。就是那种让你靠回椅背想：好，这是真实的。

大约在同一时间，我注意到另一件事：互联网开始变得更顺滑——也更糟糕。

文字变得更干净、更长、更礼貌、更……空洞。你能闻出来什么东西是生成的：信息密度低，比喻太多，太多同意，赌注不够。

我发现自己也开始这样了。

于是我给自己加了一个约束：直说。保持密度。不要膨胀。不要躲在风格后面。如果某件事重要，就把它锚定在一个真实的时刻、一个真实的权衡、一个真实付出的代价上。

因为 LLM 给不了你的不是"智慧"，而是分量。那种感觉——一个真实的人站在时间里某处，从那个位置写下了这些话。

就是在那时，我意识到我想要的不是"能说话的 AI"，而是一个能和你一起生活的 AI——安静地、持续地积累上下文，而不会把你的生活变成内容垃圾。

手机是我们最初的直觉——它私人、强大、随时在手。但移动操作系统是封闭的：没有 OEM 权限，你只能做一个 App，而不是环境基础设施。

于是我们寻找每个家庭里已经存在的常驻节点：路由器（概念上的）。但经济账不对——路由器级别的硬件承载不了记忆、RAG、工具和多用户 Agent。设备开始演化：更多内存和存储、一块屏幕、麦克风和扬声器、出门时用的小电池，可携带的形态。

最终它不再是路由器，而成为一个新品类：家庭 Agent 基础层。

## 是什么

Memoh 是一个容器化的家庭/工作室 AI 基础层：云端级别的模型能力，搭配本地优先的记忆系统（知识库、RAG/检索、对话历史），一切都在你的掌控之下。

## 为什么

长上下文模型提升了 Agent 的上限——但它同时让"完全本地"变得昂贵，让"完全云端"令人不安。人们不想每天重新给 AI 做简介，也不想让自己的持久上下文被困在别人的数据孤岛里。容器化让 Memoh 具备可移植性、可复现性，并且可以安全地作为常驻基础设施运行——让连续性变得廉价、私密且可靠。

## 怎么做

我们将 Memoh 作为容器化技术栈运行：存储（文件/数据库/向量索引）、检索、工具/运行时执行以及控制平面各自作为独立服务隔离。需要前沿能力时，推理调用云端 API；持久记忆和索引留在本地。设备充当常驻节点（类路由器，但不是路由器），为多个用户提供服务，并有严格的边界：共享是显式的，私有上下文保持私有，一切都可以作为版本化容器部署和升级。

## 功能

- **多 Bot 管理**：创建多个 Bot；人与 Bot、Bot 与 Bot 之间可以私聊、群聊或协作。

  ![多 Bot 管理](/blogs/2026-02-16/01-multi-bots.png)

- **容器化**：每个 Bot 在自己的隔离容器中运行。Bot 可以在容器内自由执行命令、编辑文件和访问网络——就像拥有自己的电脑。

  ![容器化](/blogs/2026-02-16/02-containerized.png)

- **记忆工程**：每次聊天都存储在数据库中，默认加载最近 24 小时的上下文。每轮对话都作为记忆存储，Bot 可以通过语义搜索检索。

  ![记忆工程](/blogs/2026-02-16/03-memory-engineering.png)

- **多平台支持**：支持 Telegram、飞书等平台。
- **简单易用**：通过图形界面配置 Bot 及 Provider、模型、记忆、接入平台、MCP、技能等设置——无需编写代码即可搭建你的 AI Bot。
- **定时任务**：用 cron 表达式设定定时任务，在指定时间自动执行命令。
- 更多功能持续更新……

## 与 OpenClaw 的对比

我们有一个共同的核心信念：Memoh 和 OpenClaw 都不把 Agent 当成聊天框——我们给 LLM 一个游乐场：一个真实的环境，让它可以记忆、使用工具和迭代。

Memoh 的不同之处：

- 更轻、更快：作为家庭/工作室基础设施构建，可以部署在边缘设备上
- **默认容器化**：每个 Bot 拥有独立容器（文件/命令/网络/任务）
- **混合分层**：云端推理，本地优先的记忆和索引
- **多用户优先**：显式共享和隐私边界，支持 A2A（Agent2Agent）
- **可持续**：有经验丰富的团队，有信心持续推进和构建

## 结语

Memoh 为一件事而生：常驻连续性——一个保持在线的 AI，一个属于你的记忆。

我们把前沿推理放在云端，把持久上下文留在本地，一切都作为容器化、常驻的技术栈运行。如果你想要一个感觉不像 App、更像家庭基础设施的 Agent，这就是 Memoh 所押注的方向。

此外，我们将持续运营并永久开源 Memoh，让它成为一个有长远影响力的产品。
