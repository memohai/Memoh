## Build Commands

- `mise run setup` — Install dependencies and prepare local tooling
- `mise run dev` — Start containerized dev environment (all services)
- `mise run dev:down` — Stop dev environment
- `mise run dev:logs` — View dev logs
- `mise run dev:restart -- <service>` — Restart a specific service (server, browser, web)

## Lint & Test Commands

- `mise run lint` — Run all linters (Go + ESLint)
- `mise run lint:fix` — Run all linters with auto-fix
- `mise run lint:go` — Run golangci-lint on Go code
- `mise run lint:es` — Run ESLint on TypeScript/Vue code
- `pnpm test` — Run Vitest tests for packages/
- `go test ./internal/...` — Run Go tests
- `go test -run <TestName> ./internal/<package>` — Run a single Go test

## Code Generation Commands

- `mise run sqlc-generate` — Regenerate Go code from SQL files (run after modifying db/queries/ or db/migrations/)
- `mise run swagger-generate` — Generate Swagger/OpenAPI docs from handler annotations
- `mise run sdk-generate` — Generate TypeScript SDK from OpenAPI spec (depends on swagger-generate)
- `mise run icons-generate` — Generate Vue icon components from SVG sources

## Database Commands

- `mise run db-up` — Initialize and migrate database
- `mise run db-down` — Roll back database migrations

## Architecture Overview

Memoh is a multi-bot AI agent platform with three core services:

| Service | Tech Stack | Port | Description |
|---------|-----------|------|-------------|
| **Server** | Go + Echo | 8080 | REST API, auth, database, container management, in-process AI agent |
| **Web** | Vue 3 + Vite | 8082 | Management UI for bots, models, channels |
| **Browser Gateway** | Bun + Elysia + Playwright | 8083 | Browser automation service for bots |

Infrastructure: PostgreSQL (relational), Qdrant (vector DB for memory), Containerd (container runtime).

### Project Structure

```
cmd/                        # Go application entry points
  agent/                    #   Main backend server (FX wiring)
  bridge/                   #   In-container gRPC bridge (UDS-based)
  mcp/                      #   MCP stdio transport binary
  memoh/                    #   Unified binary wrapper (Cobra)
internal/                   # Go backend core code
  agent/                    #   In-process AI agent (Twilight AI SDK)
    agent.go                #     Stream() / Generate() methods
    model.go                #     Model creation (OpenAI, Anthropic, Google)
    tools/                  #     Tool providers (message, memory, web, container, etc.)
    prompts/                #     Embedded prompt templates (system.md, etc.)
  conversation/flow/        #   Chat orchestration (resolver, streaming, memory)
  workspace/                #   Container lifecycle management
  channel/                  #   Channel adapters (Telegram, Discord, Feishu, Email)
  memory/                   #   Long-term memory system (multi-provider)
  handlers/                 #   HTTP request handlers (REST API)
  db/sqlc/                  #   Auto-generated by sqlc — DO NOT modify manually
apps/
  web/                      #   Vue 3 frontend (@memohai/web)
  browser/                  #   Browser Gateway (Bun/Elysia/Playwright)
packages/
  ui/                       #   Shared UI component library
  sdk/                      #   Auto-generated TypeScript SDK
db/
  migrations/               #   SQL migration files
  queries/                  #   SQL query files (sqlc input)
```

### Key Development Rules

**Database & sqlc:**
- SQL queries are in `db/queries/*.sql`
- All files under `internal/db/sqlc/` are auto-generated — never modify manually
- After modifying SQL, run `mise run sqlc-generate`
- Migration convention: `0001_init.up.sql` is the canonical full schema; always update it when adding schema changes, plus create incremental migration files

**API Development:**
- Write handlers in `internal/handlers/` with swaggo annotations
- Run `mise run swagger-generate` then `mise run sdk-generate` to update frontend SDK

**Agent Development:**
- AI agent runs in-process within the Go server
- Core logic in `internal/agent/`, powered by Twilight AI SDK
- Tools implement the `ToolProvider` interface in `internal/agent/tools/`
- Prompt templates are embedded Go Markdown files in `internal/agent/prompts/`

**Container/Workspace:**
- Each bot has an isolated workspace container
- Communication via gRPC bridge over Unix Domain Sockets (UDS)
- Bridge binary at `cmd/bridge/` runs inside containers

**Frontend:**
- Vue 3 Composition API with `<script setup>`
- Shared components in `packages/ui/`
- API calls via auto-generated `@memohai/sdk`
- State: Pinia + Pinia Colada; i18n: vue-i18n

## Reference Projects

### DeerFlow (Local: `/Users/solariswu/PycharmProjects/deer-flow`)

ByteDance's LangGraph-based AI agent system — useful reference for agent architecture, middleware patterns, and tool systems.

**Key Architecture Differences:**
| Aspect | Memoh | DeerFlow |
|--------|-------|----------|
| Language | Go | Python |
| Agent Framework | Twilight AI SDK (custom) | LangGraph + LangChain |
| Architecture | In-process agent | Harness/App split |
| Sandbox | Containerd per bot | Docker per thread |
| Memory | Multi-provider (Qdrant, BM25, Mem0) | JSON file + LLM extraction |
| Channels | Telegram, Discord, Feishu, Email | Telegram, Slack, Feishu |

**DeerFlow Structure:**
```
deep-flow/backend/
  packages/harness/deerflow/    # Core agent framework
    agents/lead_agent/          # Main agent with middleware chain
    agents/middlewares/         # 10 middleware components
    sandbox/                    # Docker/local sandbox execution
    subagents/                  # Sub-agent delegation
    tools/builtins/             # Built-in tools
    mcp/                        # MCP integration
    skills/                     # Skill discovery and loading
    memory/                     # Long-term memory system
  app/                          # Application layer (Gateway API, channels)
```

**DeerFlow Middleware Pattern (reference for Memoh):**
1. ThreadDataMiddleware — Per-thread directory setup
2. UploadsMiddleware — File tracking
3. SandboxMiddleware — Sandbox lifecycle
4. DanglingToolCallMiddleware — Handle interrupted tool calls
5. GuardrailMiddleware — Tool authorization
6. SummarizationMiddleware — Context reduction
7. TodoListMiddleware — Task tracking (plan mode)
8. TitleMiddleware — Auto-generate thread titles
9. MemoryMiddleware — Async memory updates
10. ViewImageMiddleware — Vision support
11. SubagentLimitMiddleware — Concurrency limits
12. ClarificationMiddleware — Handle clarification requests
